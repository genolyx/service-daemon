Client Order List

Orders on this daemon. Save stores a draft (SAVED) without starting the pipeline. Use ⋯ → Submit to queue a first run. Stop removes a draft, cancels queued, or stops a running job. ⋯ → Delete removes the order from the daemon and deletes analysis, output, and log for that run (FASTQ is kept; stop a running job first). Carrier: ⋯ → Force Run queues the full pipeline from FASTQ again (alignment + calling + post-process). Optional Reprocess only refreshes result.json/QC from existing VCFs without Nextflow.

Selected Variants for Report

When narratives fill: Values are built when this panel is shown from (1) the server gene_knowledge SQLite row for that gene when present, else (2) simple templates from panel disease + ClinVar/HGVS — that is not a live web search in the browser. Gemini runs on the server only when the cache row is missing or incomplete (Gene database → Fetch missing from Gemini) or during report generation. If you see one-line “The GENE gene is associated with …” text, the cache is empty or not configured yet. Use New search / write-up on a variant card to re-run Gemini for that gene only.

Gemini gene cache — field definitions (same as daemon extraction prompt) Search step: prioritize OMIM, GeneReviews, UniProt, GTEx; name the primary disorder. Extraction maps search text into SQLite gene_data (genetic_reporter_lookup–compatible).
  • disorder_name — Specific name of the primary disorder associated with this gene.
  • disease_association — 1–3 sentences on how pathogenic variants in this gene relate to that disorder (clinical/genetic mechanism), not just repeating the disorder name.
  • function_summary — 1–3 sentences on normal gene product function / pathway role.
  • inheritance — Pattern if known (e.g. AR, AD, XL).
  • omim_number — Digits only when cited in the text.
